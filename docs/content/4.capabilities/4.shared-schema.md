---
title: 共享类型与 schema
description: 把类型、协议和校验规则集中到 shared，避免页面和接口各写一份定义。
---

Nuva 推荐把前后端都需要的类型、接口协议和 valibot schema 放在 `shared`。这样你新增一个字段时，不需要同时去改页面校验、接口类型和服务端校验三份代码。

## 目录示例

```txt
shared/api/types.ts
shared/api/user.ts
```

## 业务类型

```ts
// shared/api/user.ts
import * as v from 'valibot'

export interface User {
  id: number
  name: string
}

export const userFormSchema = v.object({
  name: v.pipe(v.string(), v.minLength(2, '名称至少 2 个字符')),
})

export type UserFormInput = v.InferInput<typeof userFormSchema>
export type UserFormOutput = v.InferOutput<typeof userFormSchema>
```

::shared-schema-flow
::

## 使用方式

服务端和前端都从 `#shared` 引用同一份定义：

```ts
import type { User } from '#shared/api/user'
import { userFormSchema } from '#shared/api/user'
```

这样可以减少接口协议和页面类型各写一份带来的漂移。

## 什么时候不该放进 `shared`

- 只在单个页面里使用的纯展示类型。
- 只在服务端内部使用、不会暴露给前端的中间结构。
- 还没有稳定下来的临时拼装数据。
