认证相关能力统一放在这里。推荐先用 `frontend` 模式接入登录态和页面保护，只有在你要把认证协议也放进 Nuxt 项目时，再切到 Better Auth fullstack。

## 你可以在这里完成什么

- 启用 Nuva Auth 模块。
- 使用 `useAuth()` 读取登录态、跳转登录页和退出登录。
- 配置 route middleware 保护页面。
- 在请求 401 时统一清理登录态并跳转登录页。
- 启用 Better Auth fullstack 并挂载 `/api/auth/**`。

## 先做判断

- 你已经有登录接口，只需要前端读取 token、做登录跳转和页面保护：用 `frontend`。
- 你希望认证协议也由 Nuxt 项目接管，并使用 Better Auth：用 `fullstack`。

## 和请求体系的关系

业务请求仍然走 [请求体系](/request) 中的 alova 和 `useHttpClient()`。认证模块只负责登录态、跳转和认证协议；当请求返回 401 时，你可以在 `app/utils/http/hooks.ts` 中接入 `useAuth()`。
