认证相关能力统一放在这里。推荐先用 `@oevery/nuva/auth` 接入登录态和页面保护，只有在你要把认证协议也放进 Nuxt 项目时，再启用 `@oevery/nuva/better-auth`。

## 你可以在这里完成什么

- 启用 Nuva Auth 核心模块。
- 使用 `useAuth()` 读取登录态、跳转登录页和退出登录。
- 使用 `usePermission()` 做角色、功能权限、作用域和数据范围判断。
- 使用 `NuvaCan` 和服务端 `requireNuvaPermission()` 快速保护局部 UI 与 API。
- 配置 route middleware 保护页面。
- 在请求 401 时统一清理登录态并跳转登录页。
- 启用 Better Auth 可选模块并挂载 `/api/auth/**`。

## 先做判断

- 你已经有登录接口，只需要前端读取 token、做登录跳转和页面保护：用 `@oevery/nuva/auth`。
- 你希望认证协议也由 Nuxt 项目接管，并使用 Better Auth：用 `@oevery/nuva/better-auth`。

## 和请求体系的关系

业务请求仍然走 [请求体系](/request) 中的 alova 和 `useHttpClient()`。认证模块只负责登录态、跳转和认证协议桥接；当请求返回 401 时，你可以在 `app/utils/http/hooks.ts` 中接入 `useAuth()`。
