---

Auth 是 Nuva 的可选能力。项目需要登录态时，在 `nuxt.config.ts` 中加入 `@oevery/nuva/auth`。如果你还没有决定认证方案，先从 `frontend` 模式开始更稳妥。

## 启用模块

```ts
export default defineNuxtConfig({
  modules: ['@oevery/nuva/auth'],
  nuvaAuth: {
    mode: 'frontend',
    loginPath: '/login',
    homePath: '/',
    global: true,
    publicRoutes: ['/login'],
  },
})
```

## 模式

- `frontend`：提供 `useAuth()`、登录跳转和 route middleware，登录接口由项目自己实现。
- `fullstack`：在 `frontend` 能力基础上挂载 Better Auth handler。

## 什么时候选 `frontend`

- 你已有后端登录接口。
- 你只想统一 token、跳转和页面保护。
- 你不想让 `/api/auth/**` 成为项目里的核心协议入口。

## 什么时候选 `fullstack`

- 你准备使用 Better Auth 管理认证协议。
- 你希望认证逻辑和 Nuxt server API 放在一起维护。
- 你可以接受业务 API 与认证 API 分开走两套客户端入口。

## 推荐入口

应用层优先使用 `useAuth()`：

```ts
const auth = useAuth()

if (!auth.isAuthenticated.value) {
  await auth.toLogin()
}
```

## `useAuth()` 提供什么

- `mode`
- `user`
- `ready`
- `isAuthenticated`
- `logout`
- `toLogin`
- `afterLogin`
