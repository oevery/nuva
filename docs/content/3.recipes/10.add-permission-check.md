---
title: 新增权限判断
description: 在页面、局部 UI 和服务端 API 中增加统一权限判断。
---

## 目标

同时保护页面、按钮和服务端接口，避免只做前端隐藏导致接口被绕过。

## 页面级权限

```ts
definePageMeta({
  auth: {
    permissions: ['profile:update'],
  },
})
```

## 局部 UI 权限

```vue
<NuvaCan permission="profile:update">
  <UButton>保存</UButton>
</NuvaCan>
```

异步权限确认期间可以展示 pending：

```vue
<NuvaCan permission="project:create">
  <UButton>新建项目</UButton>

  <template #pending>
    <UButton disabled>
      权限确认中
    </UButton>
  </template>

  <template #fallback>
    <UButton disabled>
      无权限
    </UButton>
  </template>
</NuvaCan>
```

## 组合式判断

```ts
const permission = usePermission()

const canUpdate = computed(() => permission.can('profile:update'))
const allowed = await permission.canAsync('project:update', {
  target: {
    projectId,
  },
})
```

## 服务端 API 权限

```ts
import { defineNuvaPermissionHandler } from '@oevery/nuva/server/utils/permission'

export default defineNuvaPermissionHandler({
  permission: 'profile:update',
}, (event, auth, permission) => {
  return updateProfile(event, auth)
})
```

动态资源权限可以传 context：

```ts
export default defineNuvaPermissionHandler({
  permission: 'project:update',
  context: event => ({
    target: {
      projectId: getRouterParam(event, 'id'),
    },
  }),
}, (event, auth) => {
  return updateProject(event, auth)
})
```

## 权限字典

大型项目可以集中维护权限点：

```ts
import { defineNuvaPermissions } from '@oevery/nuva/auth/permission'

export const permissions = defineNuvaPermissions({
  profileRead: 'profile:read',
  profileUpdate: 'profile:update',
})
```

## 常见问题

- 页面级权限只负责路由能否进入。
- 按钮隐藏不是安全边界。
- 服务端 API 必须做真实鉴权。
- Better Auth 动态权限建议使用 `resource:action` 格式，例如 `project:update`。
