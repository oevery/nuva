---
title: 启用 Better Auth
description: 在需要完整认证协议时启用 Better Auth adapter，并保持业务 API 与认证 API 边界清晰。
---

## 目标

启用 `@oevery/nuva/better-auth`，让 Better Auth 负责 `/api/auth/**`，业务页面继续使用 Nuva 的 `useAuth()` 和 `usePermission()`。

## 启用模块

```ts
export default defineNuxtConfig({
  modules: ['@oevery/nuva/better-auth'],
})
```

## 配置 Nuva

```ts
// nuva.config.ts
import { defineNuvaConfig } from '@oevery/nuva/config'

export default defineNuvaConfig({
  auth: {
    provider: 'better-auth',
    betterAuth: {
      basePath: '/api/auth',
      serverAuthImport: '~~/server/utils/better-auth',
      organization: {
        hasPermission: true,
      },
    },
  },
})
```

## 提供 Better Auth 实例

业务项目需要提供 `serverAuthImport` 指向的 auth 实例，默认是：

```txt
server/utils/better-auth.ts
```

Better Auth 的邮箱登录、OAuth、organization、teams、admin、custom permissions 等能力仍按 Better Auth 文档配置。

## 客户端调用

Better Auth 协议请求使用官方 client，不走 alova：

```ts
const authClient = useBetterAuthClient()
const { data: session } = authClient.useSession()

await authClient.signIn.email({
  email,
  password,
})
```

## 和业务 API 的边界

- `/api/auth/**`：走 `useBetterAuthClient()`。
- 业务 API：走 alova 和 `useHttpClient()`。
- 业务页面：读取登录态用 `useAuth()`，权限判断用 `usePermission()`。

## 生产安全基线

至少设置：

```bash
BETTER_AUTH_SECRET="replace-with-32-chars-or-more"
BETTER_AUTH_URL="https://app.example.com"
BETTER_AUTH_TRUSTED_ORIGINS="https://app.example.com"
```

不要把模板 demo auth 作为生产认证方案。
