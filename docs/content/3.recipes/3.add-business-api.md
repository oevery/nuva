---
title: 新增业务 API 封装
description: 在 app/composables/apis 中封装业务请求，让页面不直接拼请求细节。
---

## 目标

把业务请求集中到 `app/composables/apis`，页面只调用稳定的 composable。

## 文件位置

```txt
app/composables/apis/user.ts
shared/api/user.ts
```

## 步骤

### 1. 新增 API composable

```ts
// app/composables/apis/user.ts
import type { User, UserFormInput } from '#shared/api/user'

export function useUserApi() {
  const http = useHttpClient()

  return {
    list: () => http.Get<User[]>('/users'),
    create: (data: UserFormInput) => http.Post<User>('/users', data),
  }
}
```

### 2. 页面调用

```vue
<script setup lang="ts">
const userApi = useUserApi()

const users = await userApi.list()
</script>
```

### 3. 单接口覆盖 meta

```ts
http.Get('/public-profile', {
  meta: {
    ignoreToken: true,
    envelopeUnwrap: true,
    successCodes: '0,200',
  },
})
```

## 组织建议

- 一个业务域一个文件，例如 `user.ts`、`order.ts`。
- `useXxxApi()` 只负责接口方法。
- 列表分页、筛选状态可以继续封装成 `useXxxTable()`。
- 页面不要重复处理 baseURL、响应解包、token 和错误提示。
