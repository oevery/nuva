---
title: 新增业务 API 封装
description: 在 app/composables/apis 中封装业务请求，让页面不直接拼请求细节。
---

## 目标

把业务请求集中到 `app/composables/apis`，页面只调用稳定的 composable。
模板内置了两个起步 helper：

- `createResourceApi()`：生成常见 CRUD 方法，也支持单个接口 path/method 覆盖。
- `usePageApi()`：封装分页加载、`loading`、`page`、`pageSize`、`total` 等状态。

## 文件位置

```txt
app/composables/apis/user.ts
shared/api/user.ts  # BFF/full-stack 共享 schema 时使用
```

## 步骤

### 1. 新增共享类型

```ts
// shared/api/user.ts
import * as v from 'valibot'

export interface User {
  id: string
  name: string
  status: 'enabled' | 'disabled'
}

export interface UserQuery {
  keyword?: string
  status?: User['status']
}

export const userFormSchema = v.object({
  name: v.pipe(v.string('请输入名称'), v.trim(), v.minLength(2, '名称至少 2 个字符')),
  status: v.picklist(['enabled', 'disabled'], '请选择状态'),
})

export type UserCreateInput = v.InferInput<typeof userFormSchema>
export type UserUpdateInput = Partial<UserCreateInput>
```

### 2. 新增标准 CRUD API

```ts
// app/composables/apis/user.ts
import type { User, UserCreateInput, UserQuery, UserUpdateInput } from '#shared/api/user'
import { reactive } from 'vue'

export function useUserApi() {
  return createResourceApi<User, UserCreateInput, UserUpdateInput, UserQuery>({
    path: '/users',
  })
}

export function useUserTable() {
  const query = reactive<UserQuery>({
    keyword: undefined,
    status: undefined,
  })
  const userApi = useUserApi()

  return {
    query,
    ...usePageApi<User, UserQuery>(userApi.list, {
      query,
    }),
  }
}
```

默认生成这些方法：

| 方法               | 默认请求            |
| ------------------ | ------------------- |
| `list(params)`     | `GET /users`        |
| `get(id)`          | `GET /users/:id`    |
| `create(data)`     | `POST /users`       |
| `update(id, data)` | `PATCH /users/:id`  |
| `remove(id)`       | `DELETE /users/:id` |

### 3. 覆盖非标准接口

国内后台常见接口不一定是标准 REST，可以只覆盖不标准的部分。`path` 只作为默认 REST 前缀使用：

```ts
export function useUserApi() {
  return createResourceApi<User, UserCreateInput, UserUpdateInput, UserQuery>({
    path: '/users',
    endpoints: {
      list: '/user/page',
      get: id => `/user/detail/${id}`,
      create: '/user/add',
      update: {
        path: id => `/user/save/${id}`,
        method: 'PUT',
      },
      remove: {
        path: id => `/user/remove/${id}`,
        method: 'POST',
      },
    },
  })
}
```

只有 path 不标准时可以用字符串或函数简写；path 和 method 都不标准时，用对象写到同一个 action 下。

如果接口完全不在同一个资源前缀下，可以不写根 `path`，每个 action 都在 `endpoints` 中声明：

```ts
export function useUserApi() {
  return createResourceApi<User, UserCreateInput, UserUpdateInput, UserQuery>({
    endpoints: {
      list: '/member/search',
      get: id => `/account/info/${id}`,
      create: {
        path: '/staff/new',
        method: 'POST',
      },
      update: {
        path: id => `/user/save/${id}`,
        method: 'PUT',
      },
      remove: {
        path: id => `/permission/user/${id}/delete`,
        method: 'POST',
      },
    },
  })
}
```

这种写法下，如果调用了某个没有配置 endpoint 的动作，会在运行时报出明确错误。

删除接口如果有返回数据，可以通过第 5 个泛型声明返回类型：

```ts
interface DeleteResult {
  affected: number
}

createResourceApi<User, UserCreateInput, UserUpdateInput, UserQuery, DeleteResult>({
  path: '/users',
})
```

特殊业务动作继续手写，不需要塞进 helper：

```ts
export function useUserApi() {
  const api = createResourceApi<User, UserCreateInput, UserUpdateInput, UserQuery>({
    path: '/users',
  })
  const http = useHttpClient()

  return {
    ...api,
    enable: (id: string) => http.Post<User>(`/users/${id}/enable`),
    disable: (id: string) => http.Post<User>(`/users/${id}/disable`),
  }
}
```

## 组织建议

- 一个业务域一个文件，例如 `user.ts`、`order.ts`。
- `useXxxApi()` 只负责接口方法，并返回 alova Method。
- `useXxxTable()` 负责分页、筛选、删除等列表状态。
- 标准 CRUD 用 `createResourceApi()`，特殊接口直接手写。
- 标准分页用 `usePageApi()`，特殊列表可以直接使用 alova `usePagination()`。
- `usePageApi()` 默认会过滤 `undefined`、`null` 和空字符串查询参数。
- 页面不要重复处理 baseURL、响应解包、token 和错误提示。
