---
title: 自定义响应协议
description: 调整 baseURL、响应解包、成功码和单接口 meta 以适配现有后端协议。
---

## 目标

让 Nuva 请求层适配已有后端协议，而不是在每个页面单独兼容。

## 调整全局协议

```ts
import { defineNuvaConfig } from '@oevery/nuva/config'

export default defineNuvaConfig({
  api: {
    baseURL: 'https://api.example.com/admin',
    envelopeUnwrap: true,
    successCodes: '0,200,SUCCESS',
  },
})
```

## 不需要响应解包

如果后端直接返回业务数据：

```ts
export default defineNuvaConfig({
  api: {
    envelopeUnwrap: false,
  },
})
```

## 单接口覆盖

```ts
http.Get('/legacy/report', {
  meta: {
    envelopeUnwrap: false,
    responseType: 'raw',
  },
})
```

## 跳过 token 注入

```ts
http.Post('/login', data, {
  meta: {
    ignoreToken: true,
  },
})
```

## 常见问题

- 优先在 `nuva.config.ts` 和 `app/utils/http/hooks.ts` 适配协议。
- 不要在每个页面写 `response.data.data`。
- 如果不同业务域协议差异很大，可以在对应 API composable 内集中适配。
