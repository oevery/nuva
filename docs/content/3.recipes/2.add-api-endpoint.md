title: 新增服务端接口
description: 在 server/api 中新增接口，并复用 shared 类型和 schema。
---

## 目标

新增一个 Nuxt server API，返回统一响应结构，并尽量复用 `shared` 中的类型和 schema。

## 前置条件

- 已经启动 Nuva template。
- 已经在 `shared/api` 中定义业务类型。

## 文件位置

```txt
shared/api/user.ts
server/api/users.get.ts
server/utils/response.ts
```

## 步骤

### 1. 定义 shared 类型

```ts
// shared/api/user.ts
export interface User {
  id: number
  name: string
}
```

### 2. 新增接口

```ts
// server/api/users.get.ts
import type { User } from '#shared/api/user'
import { ok } from '../utils/response'

export default defineEventHandler(() => ok<User[]>([
  { id: 1, name: 'Admin' },
]))
```

### 3. 需要校验请求体时使用 schema

```ts
// server/api/users.post.ts
import * as v from 'valibot'
import { userFormSchema } from '#shared/api/user'
import { ok } from '../utils/response'

export default defineEventHandler(async (event) => {
  const result = v.safeParse(userFormSchema, await readBody(event))

  if (!result.success) {
    throw createError({
      statusCode: 400,
      statusMessage: 'Validation Error',
      message: result.issues[0]?.message || '参数校验失败',
    })
  }

  return ok(result.output)
})
```

## 常见问题

- 不要在 server API 中重新定义一份和前端重复的类型。
- 不要只在前端做校验，服务端 API 仍必须校验输入。
- 如果接口需要登录或权限，继续看 [新增权限判断](/recipes/add-permission-check)。
