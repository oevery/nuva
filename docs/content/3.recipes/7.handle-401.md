---
title: 处理 401 和 403
description: 在请求 hooks 中统一处理登录过期、无权限提示和页面跳转。
---

## 目标

使用 `useDefaultHttpRequestHooks()` 保持统一的 401/403 行为，同时允许项目补充 toast、租户头和 traceId。

## 默认行为

- `401`：调用 `useAuth().logout()` 清理登录态，并跳转登录页。
- `403`：默认提示无权限，不跳转。
- 其它错误：按统一错误提示处理。

## 扩展请求 hooks

```ts
// app/utils/http/hooks.ts
import { useDefaultHttpRequestHooks } from '@oevery/nuva/http'

export function useHttpRequestHooks(config) {
  const defaults = useDefaultHttpRequestHooks(config, {
    notify({ message }) {
      const toast = useToast()
      toast.add({ title: message, color: 'error' })
    },
  })

  return {
    ...defaults,
    beforeRequest(method) {
      defaults.beforeRequest(method)
      method.config.headers = {
        ...(method.config.headers || {}),
        'x-tenant-id': 'tenant-demo',
      }
    },
  }
}
```

## 单接口错误文案

```ts
http.Patch('/profile', data, {
  meta: {
    errorMessage: '你没有编辑资料的权限',
  },
})
```

## 403 时跳转

页面首屏必要接口如果无权限就应该离开当前页面，可以声明：

```ts
http.Get('/profile', {
  meta: {
    forbiddenBehavior: 'redirect',
  },
})
```

## 静默请求

```ts
http.Get('/notifications/count', {
  meta: {
    forbiddenBehavior: 'silent',
    errorSilent: true,
  },
})
```

## 常见问题

- 如果你完全接管 `onSuccess` 或 `onError`，需要继续调用默认 hooks 或复用 `handleAuthHttpError()`。
- 不要在每个页面单独处理 401 跳转。
- 认证协议接口 `/api/auth/**` 不应该再额外写 alova wrapper。
