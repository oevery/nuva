---
title: 接入 OpenAPI 生成
description: 使用 @alova/wormhole 生成前端 API client，并接入 Nuva 请求体系。
---

## 目标

OpenAPI 生成属于开发期增强，不属于 Nuva core 运行时。推荐使用可选模块 `@oevery/nuva/openapi` 初始化配置，再通过 alova CLI 生成 API。

## 安装

```bash
pnpm add @alova/wormhole -D
```

## 启用模块

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  modules: [
    '@oevery/nuva/openapi',
  ],
  nuvaOpenapi: {
    input: 'http://localhost:3000/openapi.json',
    output: 'app/api/generated',
  },
})
```

模块会生成：

```txt
alova.config.ts
app/api/useGeneratedApis.ts
```

启用模块后可以通过 `#nuva/openapi` 使用生成入口：

```ts
import { useGeneratedApis } from '#nuva/openapi'
```

如果还没有执行 `alova gen`，该入口会保持可编译，但运行时调用会提示先生成 API。

## 生成 API

```bash
pnpm gen:api
```

生成结果推荐放在：

```txt
app/api/generated/
```

不要放进 `shared/api`。`shared/api` 只用于 Nuxt server 和前端真正共享的 schema/type。

## 页面状态组合

生成的 API 负责 typed alova Method，分页和页面状态仍由 Nuva helper 组合：

```ts
export function useUserTable() {
  const query = reactive<UserQuery>({
    keyword: undefined,
  })
  const apis = useGeneratedApis()

  return {
    query,
    ...usePageApi<User, UserQuery>(
      params => apis.user.listUsers({ params }),
      { query },
    ),
  }
}
```

## 边界

- Nuva 不自研 OpenAPI 解析器。
- OpenAPI 生成代码放 `app/api/generated`，不要手改。
- 人工业务包装放 `app/composables/apis` 或 `app/api/modules`。
- BFF/full-stack 共享类型才放 `shared/api`。
