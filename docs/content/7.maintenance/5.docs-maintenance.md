---
title: Docs 维护
description: 维护 Nuva 文档站的信息架构和内容边界。
---

文档站必须区分用户视角和维护者视角。

## 用户文档

用户文档包括：

- 快速开始。
- 指南。
- 配方。
- 参考。
- 部署。
- AI 辅助。

这些页面应围绕“如何使用 Nuva 写业务项目”组织。

## 维护文档

维护文档位于 `/maintenance`。只放维护 Nuva 仓库本身需要的内容：

- core/template/docs 边界。
- monorepo 命令。
- 发布和升级流程。
- 文档 IA 维护原则。

## 页面分类规则

| 问题 | 放哪里 |
| --- | --- |
| 用户项目怎么配置、开发、部署？ | 用户文档 |
| core/template/docs 怎么维护？ | `/maintenance` |
| 公开 API 怎么用？ | `/reference` |
| 公开 API 怎么变更？ | `/maintenance` |
| 具体业务任务怎么做？ | `/recipes` |

## 检查命令

```bash
pnpm docs:typecheck
pnpm docs:build
pnpm docs:generate
```

## 文档重构检查

- 旧链接是否已经替换。
- 首页路径卡片是否仍准确。
- 侧边栏标题是否面向正确受众。
- Reference 是否没有混入维护说明。
- 开发维护是否没有成为用户必读路径。
