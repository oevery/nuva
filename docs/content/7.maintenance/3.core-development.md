title: Core 开发
description: 维护 @oevery/nuva core package 时的边界和检查事项。
---

`core` 是 Nuva 的公共能力层。修改 core 前应先判断这是不是跨项目通用能力。

## 可以放进 core 的内容

- Nuxt layer 和 Nuxt module。
- 请求、认证、权限、server utils 这类基础能力。
- 可复用配置类型和默认值。
- 多个项目都会用到的 composable 或 runtime 工具。
- 公开导出入口。

## 不应该放进 core 的内容

- 某个业务项目的页面。
- 某个业务项目的具体接口。
- 某个业务项目的权限点字典。
- 为单一项目临时写的兼容逻辑。

## 修改公开 API

修改 `core/package.json` exports、配置字段或公开类型时，需要同步：

1. [导出入口](/reference/exports)。
2. [Nuva Config](/reference/nuva-config)。
3. 相关 Guide 或 Recipe。
4. Template 示例，如果用户需要按新方式使用。

## 检查命令

```bash
pnpm core:prepare
pnpm core:typecheck
pnpm core:test
```

发布前至少跑：

```bash
pnpm core:check
pnpm core:test
```

## 破坏性变更

以下变更应视为破坏性变更：

- 删除或重命名公开导出路径。
- 删除或重命名配置字段。
- 修改默认认证、请求或权限行为。
- 修改 template 必须同步的运行时约定。

破坏性变更需要更新 [升级策略](/maintenance/upgrade)。
