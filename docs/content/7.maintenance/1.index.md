## 维护原则

- 用户文档只保留使用 Nuva 写业务项目需要的内容。
- `core` 只放跨项目可复用的能力。
- `template` 保持可下载、可启动、可替换。
- `docs` 按用户视角和维护者视角分区，不混在同一页。
- 新增公开 API 时同时更新 Reference。
