---
title: 仓库结构
description: 理解 Nuva monorepo 中 core、template 和 docs 的边界。
---

Nuva 仓库是一个 pnpm workspace，核心目录如下：

```txt
core/       # @oevery/nuva package，Nuxt layer 和公共模块
template/   # 业务项目模板，可通过 giget 下载
docs/       # 文档站，面向用户和维护者
```

## `core`

`core` 是可发布包，面向多个业务项目复用。它应该包含：

- Nuxt layer 入口。
- 基础 Nuva module。
- Auth core module。
- Better Auth adapter module。
- HTTP hooks、配置类型和 server utils。

不要把单个业务项目的页面、接口和权限点写进 `core`。

## `template`

`template` 是业务起点，应该始终能直接下载运行。它应该包含：

- 推荐业务目录结构。
- 默认请求封装。
- 最小页面和表单示例。
- demo auth 或可替换认证示例。
- shared 类型和 schema 示例。

## `docs`

`docs` 是 Nuxt Content 文档站。它同时服务两类受众：

- 使用者：怎么使用 Nuva 写业务项目。
- 维护者：怎么维护 Nuva 仓库本身。

维护者内容必须放在 `/maintenance` 下，避免污染用户路径。

## 判断内容放哪里

| 内容                    | 位置      |
| ----------------------- | --------- |
| 使用 Nuva 写业务项目    | 用户文档  |
| 查询配置和公开 API      | Reference |
| 部署自己的业务项目      | 部署      |
| 维护 core/template/docs | 开发维护  |
| 发布 @oevery/nuva       | 开发维护  |
