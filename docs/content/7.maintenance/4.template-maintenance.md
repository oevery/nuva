`template` 是用户第一次接触 Nuva 的业务起点。它必须比 core 更贴近真实项目，但不能塞入过多示例噪音。

## 维护目标

- 可以通过 giget 下载并启动。
- 能展示 Nuva 的推荐目录结构。
- 保留一个完整的业务闭环。
- 认证、请求、schema、表单示例可替换。
- 不让用户误以为 demo auth 是生产方案。

## 应包含的最小闭环

- 页面路由。
- server API。
- shared 类型和 valibot schema。
- app/composables/apis 请求封装。
- 表单提交或分页列表示例。
- 登录态和页面保护示例。

## 不建议加入

- 大量视觉组件示例。
- 具体行业业务。
- 只能服务单一项目的 adapter。
- 未经过文档解释的隐藏约定。

## 检查命令

```bash
pnpm template:typecheck
pnpm template:build
pnpm template:generate
```
