---
title: 发布流程
description: 发布 core、更新 template 和部署 docs 前的检查清单。
---

## 发布 core 前

建议执行：

```bash
pnpm core:check
pnpm core:test
```

检查：

- `core/package.json` exports 是否和 [导出入口](/reference/exports) 一致。
- 新增公开 API 是否有 Reference。
- 配置字段是否有默认值和类型。
- 破坏性变更是否记录在 [升级策略](/maintenance/upgrade)。

## 更新 template 前

建议执行：

```bash
pnpm template:typecheck
pnpm template:build
```

检查：

- template 是否仍能独立下载运行。
- 默认登录、请求、schema 和页面示例是否闭环。
- demo auth 是否不会被误认为生产认证方案。
- 相关 Getting Started 或 Recipes 是否需要更新。

## 部署 docs 前

建议执行：

```bash
pnpm docs:typecheck
pnpm docs:build
pnpm docs:generate
```

检查：

- 新增页面是否出现在侧边栏。
- 内部链接是否使用新 URL。
- 用户文档和维护文档是否边界清晰。
- 首页 quick start 是否仍有效。
- `NUXT_PUBLIC_SITE_URL` 是否已设置为生产文档域名。
- `/llms.txt` 和至少一个 `/raw/...md` 端点是否可访问。

## 发布后

- 确认 npm 包版本和 git tag 一致。
- 确认文档站已部署最新内容。
- 如果有破坏性变更，确认升级说明可直接执行。
