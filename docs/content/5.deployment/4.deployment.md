---
title: 部署方式
description: 按 Nuxt 的部署模型部署 Nuva 项目，并注意认证和 runtime config。
---

Nuva 不改变 Nuxt 的部署方式。你可以按 Nuxt 应用部署到 Node、Serverless、Edge 或静态平台。

## SSR 部署

适合：

- 使用 Nuxt server API。
- 使用服务端认证或 Better Auth。
- 需要 SSR、动态数据、cookie/session。

构建后输出在 `.output`：

```bash
pnpm build
node .output/server/index.mjs
```

## 静态部署

适合：

- 页面可以预渲染。
- 不依赖 Nuxt server API 处理动态认证。
- API 由外部服务提供。

```bash
pnpm generate
```

## Better Auth 注意事项

- 必须设置生产 secret。
- 必须配置正式 `BETTER_AUTH_URL`。
- 必须配置 trusted origins。
- 需要 HTTPS 和正确的 cookie 策略。

## 反向代理注意事项

- 确认真实 IP 头只来自可信代理。
- 确认 `host`、`x-forwarded-proto` 等头不会被未信任客户端伪造。
- 登录、注册、重置密码等敏感接口保留限流。

## 部署平台

不同平台的部署细节以 Nuxt 官方文档为准。Nuva 只额外要求你确认请求、认证和 runtime config 在生产环境正确。
