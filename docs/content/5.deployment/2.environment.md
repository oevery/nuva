---
title: 环境变量
description: 配置业务项目的 API 地址、认证 secret 和生产安全变量。
---

Nuva 项目通常同时涉及 Nuxt runtime config、API 地址、认证 secret 和部署平台变量。

## API 地址

如果业务项目通过环境变量覆盖 API 地址，建议最终仍映射到 `runtimeConfig.public.nuva.api.baseURL` 或 `nuva.config.ts`。

```ts
export default defineNuxtConfig({
  runtimeConfig: {
    public: {
      nuva: {
        api: {
          baseURL: process.env.NUXT_PUBLIC_API_BASE_URL || '/api',
        },
      },
    },
  },
})
```

优先保持一个配置入口，不要让页面直接读取环境变量拼请求。

## Better Auth

生产环境至少设置：

```bash
BETTER_AUTH_SECRET="replace-with-32-chars-or-more"
BETTER_AUTH_URL="https://app.example.com"
BETTER_AUTH_TRUSTED_ORIGINS="https://app.example.com,https://admin.example.com"
```

要求：

- `BETTER_AUTH_SECRET` 至少 32 个字符。
- `BETTER_AUTH_URL` 使用正式 HTTPS 地址。
- `BETTER_AUTH_TRUSTED_ORIGINS` 不要使用过宽通配符。

## 模板 demo auth

如果保留模板 demo auth，本地开发可以设置：

```bash
NUVA_DEMO_AUTH_SECRET="replace-with-32-chars-or-more"
NUVA_DEMO_AUTH_USERNAME="admin"
NUVA_DEMO_AUTH_PASSWORD="replace-with-a-strong-password"
```

demo auth 只适合本地开发和权限链路演示。生产环境必须替换为 Better Auth、企业统一认证或真实业务认证接口。

## Secret 管理

- 不要把 `.env`、secret、私钥或生产密码提交到代码仓库。
- CI/CD 中使用部署平台的 secret store。
- 本地示例可以提交 `.env.example`，但不要填真实值。
