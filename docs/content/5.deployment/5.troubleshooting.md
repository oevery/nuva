## `nuxt prepare` 失败

优先检查：

- 依赖是否安装完整。
- `@oevery/nuva` 版本是否正确安装。
- Nuxt 版本是否满足项目要求。
- 是否在正确的业务项目根目录执行命令。

可以执行：

```bash
pnpm install
pnpm prepare
```

## 请求返回没有自动解包

检查：

- `api.envelopeUnwrap` 是否为 `true`。
- 后端是否返回 `{ code, message, data }`。
- `api.successCodes` 是否包含实际成功码。
- 单接口 `meta.envelopeUnwrap` 是否覆盖了全局配置。

## 401 没有跳转登录页

检查：

- 是否启用 `@oevery/nuva/auth`。
- `auth.enabled` 是否生效。
- 请求 hooks 是否调用了 `useDefaultHttpRequestHooks()`。
- 是否在单接口设置了 `unauthorizedBehavior: 'throw'`。

## 页面没有被保护

检查：

- `auth.global` 是 `true` 还是 `false`。
- 页面是否声明 `definePageMeta({ auth: true })`。
- 页面是否被 `auth.publicRoutes` 命中。
- 页面是否同时写了冲突的公开和受保护配置。

## 权限按钮显示异常

检查：

- `permission.source` 是否正确。
- 远程 profile 或 permission 是否返回 `roles`、`permissions`、`scope`。
- Better Auth 动态权限是否使用 `canAsync()` 或 `NuvaCan` 默认 auto 行为。
- 权限字符串是否使用 `resource:action` 格式。

## Better Auth session 不生效

检查：

- `BETTER_AUTH_SECRET`、`BETTER_AUTH_URL`、`BETTER_AUTH_TRUSTED_ORIGINS` 是否正确。
- `auth.betterAuth.basePath` 是否和 Better Auth handler 路径一致。
- SSR 请求是否通过 `useBetterAuthClient()`，不要为 `/api/auth/**` 额外写 alova wrapper。
