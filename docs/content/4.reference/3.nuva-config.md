---
title: Nuva Config
description: nuva.config.ts 的结构、字段和默认值。
---

`nuva.config.ts` 使用 `defineNuvaConfig()` 定义业务领域配置。

```ts
import { defineNuvaConfig } from '@oevery/nuva/config'

export default defineNuvaConfig({
  api: {},
  auth: {},
  resolvers: {},
})
```

## 顶层字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `api` | `Partial<NuvaApiConfig>` | 请求基础配置。 |
| `auth` | `NuvaAuthModuleOptions` | 认证、权限和菜单配置。 |
| `resolvers` | `NuvaAuthResolvers` | 自定义 profile、permission、menu resolver 路径。 |

## `api`

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `baseURL` | `/api` | API 基础地址。SSR 时相对地址会基于当前请求 origin 解析。 |
| `envelopeUnwrap` | `true` | 是否自动解包 `{ code, message, data }`。 |
| `successCodes` | `0` | 业务成功码，英文逗号分隔，例如 `0,200,SUCCESS`。 |

## `auth`

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `enabled` | `false` | 是否启用 Auth。启用 Auth module 后会打开。 |
| `provider` | `token` | 登录态 provider，可为 `token`、`better-auth` 或自定义 provider。 |
| `loginPath` | `/login` | 登录页路径。 |
| `homePath` | `/` | 登录后默认首页。 |
| `redirectQuery` | `redirect` | 登录回跳 query 名称。 |
| `global` | `false` | 是否默认保护所有页面。 |
| `publicRoutes` | `['/login']` | 公开路由，支持精确路径和 `/path/**`。 |
| `token` | 见下表 | token 读取和请求头配置。 |
| `permission` | 见下表 | 权限来源和匹配配置。 |
| `accessMenu` | 见下表 | 菜单来源和过滤配置。 |
| `betterAuth` | 见下表 | Better Auth adapter 配置。 |

## `auth.token`

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `cookieName` | `token` | SSR 和客户端读取 token 的 cookie 名称。 |
| `storageKey` | `token` | 客户端 localStorage 键名。 |
| `header` | `Authorization` | 发送 token 时使用的请求头名称。 |
| `prefix` | `Bearer` | token 前缀，后端需要原始 token 时传空字符串。 |

## `auth.permission`

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `source` | `local` | `local`、`remote`、`hybrid` 或 `adapter`。 |
| `forbiddenPath` | `/403` | 无权限跳转路径。 |
| `permissionMode` | `all` | 多权限默认匹配方式。 |
| `roleMode` | `any` | 多角色默认匹配方式。 |
| `local` | 空权限态 | 本地角色、权限、作用域和数据范围。 |
| `remote.profile` | 空 | 远程用户态请求配置。 |
| `remote.permission` | 空 | 远程权限请求配置。 |
| `remote.cacheMaxAge` | `0` | 路由守卫中用户态和权限态缓存时间。 |

## `auth.accessMenu`

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `source` | `none` | 菜单来源：`none`、`profile`、`remote`、`route`。 |
| `filter` | `true` | 是否按权限过滤菜单。 |
| `routePrune` | `true` | 是否移除没有对应路由的菜单。 |
| `strictRoute` | `true` | 是否要求菜单权限与路由权限一致。 |
| `cacheMaxAge` | `0` | 菜单缓存时间。 |
| `remote.menu` | 空 | 远程菜单请求配置。 |

## `auth.betterAuth`

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `basePath` | `/api/auth` | Better Auth handler 路径。 |
| `serverAuthImport` | `~~/server/utils/better-auth` | Better Auth 实例导入路径。 |
| `organization.enabled` | `false` | 是否启用 Better Auth organization client。 |
| `organization.hasPermission` | `false` | 是否使用 organization `hasPermission` 做动态权限。 |
| `organization.dynamicAccessControl` | `false` | 是否启用 dynamic access control client。 |
