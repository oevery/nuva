---
title: Nuva Config
description: nuva.config.ts 的结构、字段和默认值。
---

`nuva.config.ts` 使用 `defineNuvaConfig()` 定义业务领域配置。

```ts
import { defineNuvaConfig } from '@oevery/nuva/config'

export default defineNuvaConfig({
  api: {},
  auth: {},
  resolvers: {},
})
```

## 顶层字段

| 字段        | 类型                    | 说明                                          |
| ----------- | ----------------------- | --------------------------------------------- |
| `api`       | `NuvaApiConfigOptions`  | 请求基础配置。                                |
| `auth`      | `NuvaAuthModuleOptions` | 认证、权限和菜单配置。                        |
| `resolvers` | `NuvaAuthResolvers`     | 自定义 user、permission、menu resolver 路径。 |

## `api`

| 字段             | 默认值 | 说明                                                     |
| ---------------- | ------ | -------------------------------------------------------- |
| `baseURL`        | `/api` | API 基础地址。SSR 时相对地址会基于当前请求 origin 解析。 |
| `envelopeUnwrap` | `true` | 是否自动解包 `{ code, message, data }`。                 |
| `successCodes`   | `0`    | 业务成功码，英文逗号分隔，例如 `0,200,SUCCESS`。         |

### `api.response`

| 字段         | 默认值    | 说明             |
| ------------ | --------- | ---------------- |
| `codeKey`    | `code`    | 业务状态码路径。 |
| `messageKey` | `message` | 消息字段路径。   |
| `dataKey`    | `data`    | 数据字段路径。   |

### `api.pagination`

| 字段              | 默认值     | 说明                                      |
| ----------------- | ---------- | ----------------------------------------- |
| `pageField`       | `pageNum`  | 请求页码字段。                            |
| `pageSizeField`   | `pageSize` | 请求每页数量字段。                        |
| `listKey`         | `list`     | 响应列表字段路径。                        |
| `totalKey`        | `total`    | 响应总数字段路径。                        |
| `initialPage`     | `1`        | 初始页码。                                |
| `initialPageSize` | `10`       | 初始每页数量。                            |
| `cleanParams`     | `true`     | 是否过滤 `undefined`、`null` 和空字符串。 |

Nuva 会在 prepare/build 阶段根据静态 `api.response` 和 `api.pagination` 生成 `#nuva/api` 类型入口。非标字段名如 `current-page` 会生成字符串字面量属性；复杂路径如 `data.records` 运行时仍支持读取，但生成类型会回退到默认字段名，复杂结构建议在业务 API 中用显式响应类型和 mapper 适配。

## `auth`

| 字段            | 默认值       | 说明                                                             |
| --------------- | ------------ | ---------------------------------------------------------------- |
| `enabled`       | `false`      | 是否启用 Auth。启用 Auth module 后会打开。                       |
| `provider`      | `token`      | 登录态 provider，可为 `token`、`better-auth` 或自定义 provider。 |
| `loginPath`     | `/login`     | 登录页路径。                                                     |
| `homePath`      | `/`          | 登录后默认首页。                                                 |
| `redirectQuery` | `redirect`   | 登录回跳 query 名称。                                            |
| `global`        | `false`      | 是否默认保护所有页面。                                           |
| `publicRoutes`  | `['/login']` | 公开路由，支持精确路径和 `/path/**`。                            |
| `token`         | 见下表       | token 读取和请求头配置。                                         |
| `user`          | 见下表       | 当前用户远程请求配置。                                           |
| `permission`    | 见下表       | 权限来源和匹配配置。                                             |
| `accessMenu`    | 见下表       | 菜单来源和过滤配置。                                             |
| `betterAuth`    | 见下表       | Better Auth adapter 配置。                                       |

## `auth.token`

| 字段         | 默认值          | 说明                                          |
| ------------ | --------------- | --------------------------------------------- |
| `cookieName` | `token`         | SSR 和客户端读取 token 的 cookie 名称。       |
| `storageKey` | `token`         | 客户端 localStorage 键名。                    |
| `header`     | `Authorization` | 发送 token 时使用的请求头名称。               |
| `prefix`     | `Bearer`        | token 前缀，后端需要原始 token 时传空字符串。 |

## `auth.user`

| 字段                 | 默认值 | 说明                                  |
| -------------------- | ------ | ------------------------------------- |
| `remote.request`     | 空     | 当前用户请求配置，常用于 `/getInfo`。 |
| `remote.map`         | `null` | 从响应中映射当前用户的路径或字段表。  |
| `remote.cacheMaxAge` | `0`    | 当前用户缓存时间。                    |

## `auth.permission`

| 字段                 | 默认值   | 说明                                                                      |
| -------------------- | -------- | ------------------------------------------------------------------------- |
| `source`             | `local`  | `local`、`remote` 或 `adapter`。                                          |
| `forbiddenPath`      | `/403`   | 无权限跳转路径。                                                          |
| `permissionMode`     | `all`    | 多权限默认匹配方式。                                                      |
| `roleMode`           | `any`    | 多角色默认匹配方式。                                                      |
| `local`              | 空权限态 | 本地角色、权限、作用域和数据范围。                                        |
| `remote.request`     | 空       | 远程权限请求配置，可与 `auth.user.remote.request` 指向同一个 `/getInfo`。 |
| `remote.map`         | `null`   | 从响应中映射 `roles`、`permissions`、`scope` 和 `dataAccess`。            |
| `remote.cacheMaxAge` | `0`      | 权限态缓存时间。                                                          |

## `auth.accessMenu`

| 字段                   | 默认值  | 说明                                                 |
| ---------------------- | ------- | ---------------------------------------------------- |
| `source`               | `route` | 菜单来源：`route`、`local`、`remote` 或 `adapter`。  |
| `adapter`              | 空      | access menu adapter 名称。                           |
| `visibility`           | `route` | 菜单显示策略：`route`、`backend`、`menu`、`strict`。 |
| `cacheMaxAge`          | `0`     | 菜单缓存时间。                                       |
| `route.mode`           | `mixed` | 路由菜单模式：`auto`、`meta` 或 `mixed`。            |
| `route.includeDynamic` | `false` | 是否把动态路由加入自动菜单。                         |
| `route.exclude`        | `[]`    | 路由菜单排除路径。                                   |
| `local.items`          | `[]`    | 本地静态菜单。                                       |
| `remote.request`       | 空      | 远程菜单请求配置。                                   |
| `remote.map`           | `null`  | 从响应中映射菜单列表路径。                           |

`adapter` 为空时，`source: 'adapter'` 会跟随当前 auth provider。

`visibility` 控制菜单显示由谁决定：`route` 使用 Nuxt route meta auth，`backend` 信任后端裁剪菜单，`menu` 使用菜单项自身权限字段，`strict` 同时要求菜单权限和路由权限一致。

## `auth.betterAuth`

| 字段                                | 默认值                        | 说明                                               |
| ----------------------------------- | ----------------------------- | -------------------------------------------------- |
| `basePath`                          | `/api/auth`                   | Better Auth handler 路径。                         |
| `serverAuthImport`                  | `~~/server/utils/better-auth` | Better Auth 实例导入路径。                         |
| `organization.enabled`              | `false`                       | 是否启用 Better Auth organization client。         |
| `organization.hasPermission`        | `false`                       | 是否使用 organization `hasPermission` 做动态权限。 |
| `organization.dynamicAccessControl` | `false`                       | 是否启用 dynamic access control client。           |
