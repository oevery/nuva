## `useAuth()`

```ts
const auth = useAuth()
```

返回字段：

| 字段 | 说明 |
| --- | --- |
| `provider` | 当前 auth provider。 |
| `user` | 当前用户。 |
| `ready` | 登录态是否初始化完成。 |
| `isAuthenticated` | 是否已登录。 |
| `refresh()` | 刷新登录态。 |
| `logout()` | 退出登录并清理权限与菜单缓存。 |
| `toLogin()` | 跳转登录页。 |
| `afterLogin()` | 登录后按 redirect 回跳。 |

## `useAccessMenu()`

返回字段：

| 字段 | 说明 |
| --- | --- |
| `menus` | 过滤后的菜单。 |
| `rawMenus` | 原始菜单。 |
| `ready` | 菜单是否可用。 |
| `setMenus(items)` | 写入菜单。 |
| `clearMenus()` | 清理菜单。 |
| `refresh()` | 刷新远程菜单。 |
| `ensure()` | 确保菜单已加载。 |

## Auth adapter registry

从 `@oevery/nuva/auth/adapter` 导入：

| API | 说明 |
| --- | --- |
| `defineAuthAdapter(factory)` | 定义 auth adapter factory。 |
| `registerAuthAdapter(name, factory)` | 注册客户端/通用 auth adapter。 |
| `useAuthAdapter(name)` | 读取 auth adapter。 |
| `resetAuthAdapters()` | 清理外部注册的 adapter，常用于测试。 |

## Server adapter registry

从 `@oevery/nuva/auth/server-adapter` 导入：

| API | 说明 |
| --- | --- |
| `defineServerAuthAdapter(factory)` | 定义 server adapter factory。 |
| `registerServerAuthAdapter(name, factory)` | 注册 server adapter。 |
| `useServerAuthAdapter(name)` | 读取 server adapter。 |
| `resetServerAuthAdapters()` | 清理 server adapter。 |

## Token provider

`useTokenAuthClient()` 是 token provider 专用 API，用于登录接口返回 token 后同步状态。业务页面读取登录态仍优先使用 `useAuth()`。

```ts
const tokenAuth = useTokenAuthClient<CurrentUser>()
tokenAuth.setSession(data.token, data.user)
```
