## `usePermission()`

```ts
const permission = usePermission()
```

返回字段：

| 字段                                             | 说明                                  |
| ------------------------------------------------ | ------------------------------------- |
| `authReady`                                      | 认证态是否初始化完成。                |
| `permissionReady` / `ready`                      | 权限态是否可用。                      |
| `loaded`                                         | 权限是否已加载。                      |
| `source`                                         | 权限来源。                            |
| `roles`                                          | 当前角色。                            |
| `permissions`                                    | 当前功能权限。                        |
| `scope`                                          | 当前作用域。                          |
| `dataAccess`                                     | 当前数据范围。                        |
| `state`                                          | 完整权限态。                          |
| `setPermission(permission)`                      | 设置权限态。                          |
| `refresh()`                                      | 刷新权限。                            |
| `ensure()`                                       | 确保权限可用。                        |
| `can(permission)`                                | 同步判断单个权限。                    |
| `canAsync(permission, context?)`                 | 异步判断单个权限。                    |
| `any(permissions)`                               | 任一权限匹配。                        |
| `all(permissions)`                               | 所有权限匹配。                        |
| `anyAsync(permissions, context?)`                | 异步任一权限匹配。                    |
| `allAsync(permissions, context?)`                | 异步所有权限匹配。                    |
| `hasRole(role, mode?)`                           | 判断角色。                            |
| `hasScope(scope, mode?)`                         | 判断作用域。                          |
| `canAccess(resource, action?)`                   | 使用 `resource:action` 格式判断权限。 |
| `explain()` / `explainRole()` / `explainScope()` | 返回诊断结果。                        |

## `NuvaCan`

```vue
<NuvaCan permission="profile:update">
  <UButton>保存</UButton>
</NuvaCan>
```

常用 slot：

| Slot       | 说明               |
| ---------- | ------------------ |
| `default`  | 有权限时显示。     |
| `pending`  | 异步确认期间显示。 |
| `fallback` | 无权限时显示。     |

## 权限工具

从 `@oevery/nuva/auth/permission` 导入：

| API                                              | 说明                          |
| ------------------------------------------------ | ----------------------------- |
| `defineNuvaPermissions(catalog)`                 | 定义权限字典。                |
| `matchList(owned, required, mode?)`              | 判断列表匹配。                |
| `explainList(owned, required, mode?, reason?)`   | 返回列表匹配诊断。            |
| `resolvePermissionState(user, fallback, source)` | 从用户态解析权限态。          |
| `hasPermissionState(value)`                      | 判断对象是否包含权限态。      |
| `hasScope(scope, required, mode?)`               | 判断作用域。                  |
| `explainScope(scope, required, mode?)`           | 返回作用域诊断。              |
| `toResourcePermission(resource, action?)`        | 生成 `resource:action`。      |
| `toBetterAuthPermissions(input)`                 | 转换成 Better Auth 权限对象。 |
