---
title: 导出入口
description: @oevery/nuva 的公开导入路径和用途。
---

`@oevery/nuva` 通过子路径导出稳定能力。业务项目应优先使用这些入口，不要依赖 core 内部相对路径。

| 导入路径                                  | 用途                                           |
| ----------------------------------------- | ---------------------------------------------- |
| `@oevery/nuva`                            | Nuxt layer 根入口。                            |
| `@oevery/nuva/base`                       | Nuva 基础模块入口。                            |
| `@oevery/nuva/nuxt.config`                | 显式 Nuxt 配置入口。                           |
| `@oevery/nuva/config`                     | Nuva 配置类型、默认值和 `defineNuvaConfig()`。 |
| `@oevery/nuva/nuva-config`                | `@oevery/nuva/config` 的别名。                 |
| `@oevery/nuva/http`                       | HTTP hooks、错误处理、token 工具和类型。       |
| `@oevery/nuva/auth`                       | Auth core module。                             |
| `@oevery/nuva/auth/adapter`               | Adapter registry。                             |
| `@oevery/nuva/auth/server-adapter`        | 服务端 auth adapter registry。                 |
| `@oevery/nuva/auth/permission`            | 权限工具和权限字典。                           |
| `@oevery/nuva/auth/access-menu`           | 菜单标准化和校验工具。                         |
| `@oevery/nuva/auth/state`                 | 清理 Auth 派生状态。                           |
| `@oevery/nuva/auth/token`                 | token provider client。                        |
| `@oevery/nuva/better-auth`                | Better Auth adapter module。                   |
| `@oevery/nuva/better-auth/client`         | `useBetterAuthClient()`。                      |
| `@oevery/nuva/better-auth/server-adapter` | `registerBetterAuthServerAdapter()`。          |
| `@oevery/nuva/server/utils/config`        | 服务端读取 Nuva config。                       |
| `@oevery/nuva/server/utils/permission`    | 服务端权限保护工具。                           |
| `@oevery/nuva/server/*`                   | 透出 server 下模块。                           |
| `@oevery/nuva/app/*`                      | 透出 app 下模块。                              |

## 建议

- 业务项目不要从 `@oevery/nuva/modules/...` 这类内部路径导入。
- 参考文档中出现的路径优先级高于源码内部路径。
- 如果某个能力没有公开导出，应先判断是否真的需要作为公共 API。
