## 配置工具

从 `@oevery/nuva/server/utils/config` 导入：

| API | 说明 |
| --- | --- |
| `getNuvaConfig(event)` | 从 `runtimeConfig.public.nuva` 读取并解析 Nuva 配置。 |

## 权限保护工具

从 `@oevery/nuva/server/utils/permission` 导入：

| API | 说明 |
| --- | --- |
| `setNuvaAuthContext(event, permission)` | 写入当前请求的权限上下文。 |
| `getNuvaPermissionState(event, options?)` | 读取当前请求权限态。 |
| `getNuvaAuthContext(event, provider?)` | 通过 server adapter 解析认证上下文。 |
| `requireNuvaAuthContext(event, provider?)` | 要求已登录，否则返回 401。 |
| `requireNuvaPermission(event, permission, input?)` | 同步权限保护。 |
| `requireNuvaPermissionAsync(event, permission, input?)` | 支持 adapter 动态权限的异步保护。 |
| `requireNuvaRole(event, role, input?)` | 要求角色。 |
| `requireNuvaScope(event, scope, input?)` | 要求作用域。 |
| `requireNuvaDataAccessType(event, type, options?)` | 要求数据范围类型。 |
| `requireNuvaDataAccess(event, target, options?)` | 根据目标资源判断数据范围。 |
| `defineProtectedHandler(options, handler)` | 自定义 auth 函数的声明式保护。 |
| `definePermissionHandler(options, handler)` | 自定义 auth 函数的单权限保护。 |
| `defineNuvaProtectedHandler(options, handler)` | 使用当前 provider server adapter 的声明式保护。 |
| `defineNuvaPermissionHandler(options, handler)` | 使用当前 provider server adapter 的单权限保护。 |

## 推荐入口

多数业务 API 优先使用：

```ts
import { defineNuvaPermissionHandler } from '@oevery/nuva/server/utils/permission'

export default defineNuvaPermissionHandler({
  permission: 'profile:update',
}, (event, auth, permission) => {
  return updateProfile(event, auth)
})
```

需要同时声明角色、权限和动态上下文时使用 `defineNuvaProtectedHandler()`。
