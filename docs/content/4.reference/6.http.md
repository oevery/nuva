HTTP 公开入口：

```ts
import { useDefaultHttpRequestHooks } from '@oevery/nuva/http'
```

## 导出函数

| API                                                     | 说明                                                             |
| ------------------------------------------------------- | ---------------------------------------------------------------- |
| `useDefaultHttpRequestHooks(config, options?)`          | 返回 alova hooks，处理 cookie 转发、token 注入、响应解包和错误。 |
| `handleAuthHttpError(error, config, method?, options?)` | 处理 401、403 和错误提示。                                       |
| `resolveHttpErrorMessage(error, method?)`               | 解析错误提示文案。                                               |
| `getHttpErrorStatus(error)`                             | 解析错误状态码。                                                 |
| `applyAuthHeader(method, token, config)`                | 给请求注入 token header。                                        |
| `resolveTokenValue(token, config)`                      | 解析 token 字符串和前缀。                                        |

## 导出类型

| 类型                             | 说明                                           |
| -------------------------------- | ---------------------------------------------- |
| `HttpResponseType`               | `json`、`text`、`blob`、`arrayBuffer`、`raw`。 |
| `HttpSuccessCodes`               | 字符串、数字或字符串/数字数组。                |
| `HttpForbiddenBehavior`          | `notify`、`redirect`、`silent`、`throw`。      |
| `DefaultHttpRequestHooksOptions` | 默认 hooks 选项。                              |
| `HttpErrorNotifyPayload`         | 错误提示回调参数。                             |

## Core 请求 helper

| API                       | 说明                                        |
| ------------------------- | ------------------------------------------- |
| `useNuvaPageApi()`        | 基于 alova `usePagination` 的分页状态组合。 |
| `createNuvaResourceApi()` | 生成常见资源 CRUD alova Method。            |

## alova meta 字段

| 字段                   | 说明                                       |
| ---------------------- | ------------------------------------------ |
| `envelopeUnwrap`       | 单接口是否解包 `{ code, message, data }`。 |
| `ignoreToken`          | 是否跳过 token 注入。                      |
| `responseType`         | 单接口响应解析类型。                       |
| `successCodes`         | 单接口业务成功码。                         |
| `forbiddenBehavior`    | 当前接口返回 403 时的行为。                |
| `unauthorizedBehavior` | 当前接口返回 401 时的行为。                |
| `errorMessage`         | 当前接口错误提示文案。                     |
| `errorSilent`          | 是否静默错误提示。                         |

## 示例

```ts
http.Get('/profile', {
  meta: {
    forbiddenBehavior: 'redirect',
    errorMessage: '你没有查看资料的权限',
  },
})
```
