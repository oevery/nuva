---
title: 为什么 Nuva
description: Nuva 不是 UI 组件库或低代码平台，而是一套 Nuxt 业务项目的基础约定。
---

Nuva 是一个面向 Nuxt 业务项目的轻量全栈底座。它把每个项目都会重复搭建的请求、认证、权限、共享 schema 和表单校验先整理成稳定约定，让你更快进入真实业务开发。

## Nuva 是什么

Nuva 由三部分组成：

- `core`：发布为 `@oevery/nuva` 的 Nuxt layer 和可选模块。
- `template`：可以直接下载运行的业务模板。
- `docs`：描述使用方式、设计边界、任务配方和 API 参考。

它的目标不是替你写完整业务系统，而是提供一套可复用的业务起点。

## Nuva 解决什么问题

很多 Nuxt 项目启动前都会重复做这些事：

- 配请求基础地址、响应解包、错误处理和 token 注入。
- 写登录态读取、路由保护、401 跳转和权限判断。
- 在前端表单和服务端接口里重复维护字段校验。
- 不断解释 `app`、`server`、`shared` 里应该放什么。
- 每个项目都从空白模板重新整理业务目录。

Nuva 把这些内容沉到 core layer 和 template。业务项目只需要先接受一套简单约定，再按业务逐步替换默认实现。

## Nuva 不是什么

- 不是 UI 组件库。Nuva 不规定页面视觉和业务组件体系。
- 不是低代码平台。Nuva 不负责生成完整后台页面。
- 不是认证服务。Better Auth 只是可选 adapter，真实认证协议仍由项目配置。
- 不是替代 Nuxt。Nuva 运行在 Nuxt 之上，仍遵循 Nuxt 的目录、模块和部署方式。

## 适合什么项目

- 中后台、运营工具、BFF 和 Nuxt 全栈业务应用。
- 需要统一请求协议、错误处理和登录态管理的项目。
- 希望前后端共用类型、schema 和接口协议的项目。
- 想先拿到一个能跑的模板，再逐步替换成真实业务的团队。

## 不适合什么项目

- 只需要静态展示页，不需要请求、认证或 server API。
- 团队已经有完整内部框架，且请求、认证、权限、模板都已稳定。
- 只想找 UI 组件库或后台 Pro 组件。
