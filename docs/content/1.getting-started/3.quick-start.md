本页只做一件事：把 Nuva template 跑起来，并确认默认业务闭环可用。

## 环境要求

- Node.js 20+
- `pnpm` 11+

## 下载模板

```bash
pnpm dlx giget@latest gh:oevery/nuva/template my-app
cd my-app
pnpm install
pnpm dev
```

`giget` 会把 `template` 目录下载成你的新项目。下载后你就在这个项目里继续开发，不需要保留 monorepo 结构。

## 启动后先检查

1. 首页和登录页是否能正常打开。
2. `server/api` 下的默认接口是否能响应。
3. `shared` 中的类型和 valibot schema 是否能被前端和服务端同时引用。
4. `app/composables/apis` 的默认 API 封装是否符合你的接口协议。

## 常用命令

```bash
pnpm dev
pnpm build
pnpm generate
pnpm preview
pnpm lint
pnpm typecheck
```

## 第一次接项目时不建议做什么

- 不要一开始就重做底层结构。
- 不要同时引入第二套请求方案。
- 不要把业务类型重新散落到页面和接口文件里。
- 不要在没有确认认证模式前强行接入 Better Auth。
