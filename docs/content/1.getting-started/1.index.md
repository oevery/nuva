这一组文档只服务一个目标：让你尽快判断 Nuva 是否适合当前项目，并把默认模板跑成一个可继续开发的业务起点。

## 推荐阅读顺序

1. [为什么 Nuva](/getting-started/why-nuva)：理解 Nuva 解决什么问题，以及它不是什么。
2. [快速启动](/getting-started/quick-start)：下载模板、安装依赖并启动项目。
3. [第一个业务功能](/getting-started/first-feature)：用用户列表串起 `shared -> server/api -> composables/apis -> page`。
4. [项目结构](/getting-started/project-structure)：知道页面、接口、共享类型和配置分别放哪里。
5. [基础配置](/getting-started/configuration)：理解 `nuva.config.ts` 和 `runtimeConfig.public.nuva` 的关系。
6. [下一步](/getting-started/next-steps)：按你的目标进入指南、配方、参考或部署维护。

## 第一次接入时先做什么

1. 下载并启动模板。
2. 确认首页、登录页、默认接口和 shared schema 都能正常工作。
3. 先改请求协议或认证配置，不要一开始重做目录结构。
4. 用一个真实业务功能验证 Nuva 的组织方式是否适合团队。

## 成功标准

- 你知道 Nuva 和普通 Nuxt 模板的区别。
- 你能启动 template 并找到 `app`、`server`、`shared`。
- 你知道业务接口、请求封装、页面和表单 schema 的推荐落点。
- 你能判断下一步该看 Guide、Recipes 还是 Reference。
