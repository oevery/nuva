本页目标只有一个：先把项目启动成功，并确认默认流程符合你的预期。

## 环境要求

- Node.js 20+
- `pnpm` 11+

## 下载模板

```bash
pnpm dlx giget@latest gh:oevery/nuva/template my-app
cd my-app
pnpm install
pnpm dev
```

`giget` 会把模板下载到你的项目目录，随后你就在这个项目里继续开发。

## 启动后先看什么

启动成功后，建议按这个顺序确认：

1. 首页和登录页是否能正常打开。
2. `server/api` 下的默认接口是否能响应。
3. `shared` 中的类型和 schema 是否已经能被前后端同时引用。
4. `app/composables/apis` 里的默认 API 封装是否符合你的接口协议。

## 启动项目

```bash
pnpm dev
```

默认情况下，这个命令会启动你刚创建的 Nuva 项目。

## 第一次接项目时，不建议立刻做的事

- 不要一开始就改底层结构或重做目录。
- 不要同时引入第二套请求方案。
- 不要把业务类型散落回页面和接口文件里。

## 常用命令

```bash
pnpm dev
pnpm build
pnpm generate
pnpm preview
pnpm lint
pnpm typecheck
```

## 什么时候用这些命令

- `pnpm dev`：本地开发时使用。
- `pnpm build`：检查生产构建是否通过。
- `pnpm generate`：需要静态生成时使用。
- `pnpm preview`：预览构建结果。
- `pnpm lint`：检查代码风格和常见问题。
- `pnpm typecheck`：检查类型是否正确。
