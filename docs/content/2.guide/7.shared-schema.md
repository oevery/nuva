---
title: 共享类型与 Schema
description: 把类型、协议和校验规则集中到 shared，避免页面和接口各写一份定义。
---

Nuva 推荐把前后端都需要的类型、接口协议和 valibot schema 放在 `shared`。这样新增字段时，不需要分别修改页面校验、接口类型和服务端校验三份代码。

## 目录示例

```txt
shared/api/types.ts
shared/api/user.ts
```

## 业务类型和 schema

```ts
// shared/api/user.ts
import * as v from 'valibot'

export interface User {
  id: number
  name: string
}

export const userFormSchema = v.object({
  name: v.pipe(v.string(), v.minLength(2, '名称至少 2 个字符')),
})

export type UserFormInput = v.InferInput<typeof userFormSchema>
export type UserFormOutput = v.InferOutput<typeof userFormSchema>
```

::shared-schema-flow
::

## 使用方式

前端和服务端都从 `#shared` 引用：

```ts
import type { User } from '#shared/api/user'
import { userFormSchema } from '#shared/api/user'
```

## 什么时候应该放进 `shared`

- 接口请求或响应类型。
- 服务端和前端都需要的业务实体类型。
- 前端表单和服务端接口都需要的 valibot schema。
- 通用分页、枚举和协议结构。

## 什么时候不该放进 `shared`

- 只在单个页面使用的展示类型。
- 只在服务端内部使用的中间结构。
- 还没有稳定下来的临时拼装数据。
