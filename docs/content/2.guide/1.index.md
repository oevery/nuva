Guide 层负责解释设计和边界。具体任务步骤放在 [配方](/recipes)，完整字段和 API 放在 [参考](/reference)。

## 推荐阅读顺序

1. [整体架构](/guide/architecture)：理解页面、请求、server API 和 shared schema 的协作关系。
2. [Core layer 与 Template](/guide/core-layer-and-template)：理解可复用层和业务模板的边界。
3. [App / Server / Shared](/guide/app-server-shared)：判断业务代码应该放哪里。
4. [请求体系](/guide/request-system)：理解 alova、`useHttpClient()`、响应解包和 hooks。
5. [认证与权限](/guide/auth-and-permission)：理解 Auth core、权限模型和 Better Auth adapter。
6. [共享类型与 Schema](/guide/shared-schema)：理解为什么 schema 应该放在 `shared`。
7. [表单与校验](/guide/forms-and-validation)：理解前端表单和服务端校验如何共用规则。
8. [Runtime Config](/guide/runtime-config)：理解配置如何从 `nuva.config.ts` 进入运行时。

## 和其它层级的区别

- Getting Started：让你跑起来。
- Guide：解释设计边界。
- Recipes：解决具体任务。
- Reference：完整查配置和 API。
- 部署：上线业务项目和排错。
- 开发维护：维护 Nuva 仓库本身。
