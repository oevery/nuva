Auth 是 Nuva 的可选能力。推荐先用 `@oevery/nuva/auth` 接入登录态、路由保护和权限判断；只有当你希望认证协议也由 Nuxt 项目接管时，再启用 `@oevery/nuva/better-auth`。

## 两类接入方式

| 场景                                           | 推荐方式                   |
| ---------------------------------------------- | -------------------------- |
| 已有后端登录接口，只需要 token、跳转和页面保护 | `@oevery/nuva/auth`        |
| 认证协议也放在 Nuxt 项目内维护                 | `@oevery/nuva/better-auth` |
| 企业 SSO、Auth.js、Supabase 或自研 session     | 自定义 auth adapter        |

## Auth core 负责什么

`@oevery/nuva/auth` 提供：

- `useAuth()`：登录态、用户、退出、跳转登录页和登录后回跳。
- `usePermission()`：角色、权限、作用域、数据范围和动态权限判断。
- `useAccessMenu()`：菜单来源、过滤和路由一致性校验。
- `NuvaCan`：局部 UI 权限显示。
- route middleware：页面登录保护和页面级权限保护。
- server utils：服务端 API 登录与权限保护。

## 权限模型

Nuva 把权限拆成几层：

- `roles`：角色，例如 `admin`。
- `permissions`：功能权限，推荐 `resource:action`，例如 `profile:update`。
- `scope`：作用域，例如 `tenantId`、`organizationId`、`departmentId`。
- `dataAccess`：数据范围，例如 `all`、`self`、`department`。

业务页面优先使用 `usePermission()`：

```ts
const permission = usePermission()

permission.can('profile:update')
await permission.canAsync('profile:update')
permission.any(['profile:update', 'profile:approve'])
permission.hasRole('admin')
permission.hasScope('departmentId')
```

## 权限来源

`permission.source` 支持：

- `local`：本地静态角色和权限。
- `remote`：远程接口返回角色、权限、作用域和数据范围。
- `adapter`：由当前 auth provider 的 adapter 提供权限态和动态权限校验。

多数已有后端的后台项目可以先用 `auth.user.remote.request` 和 `permission.remote.request` 指向同一个 `/getInfo`。Nuva 会按请求配置去重，并通过 `remote.map` 从同一份响应里映射权限和菜单。

## 页面保护

`auth.global: false` 时，只有显式声明 `definePageMeta({ auth: true })` 或声明权限的页面受保护。

`auth.global: true` 时，页面默认受保护，公开页面需要声明：

```ts
definePageMeta({
  auth: false,
})
```

页面权限可以声明：

```ts
definePageMeta({
  auth: {
    roles: ['admin'],
    permissions: ['profile:read'],
    scopes: ['departmentId'],
  },
})
```

## Better Auth 的边界

Better Auth 是可选 adapter module，不是 Nuva 权限系统本身。启用后：

- `/api/auth/**` 交给 Better Auth handler。
- Better Auth 协议请求走 `useBetterAuthClient()`。
- 业务 API 仍走 alova 和 `useHttpClient()`。
- 业务页面仍优先消费 `useAuth()` 和 `usePermission()`。
